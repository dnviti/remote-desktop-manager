[~] GATE-038 — Routing SSH tramite bastion/jump host
------------------------------------------------------------------------------
  Priorita: ALTA
  Dipendenze: GATE-036

  DESCRIZIONE:
  Instradare le connessioni SSH attraverso un bastion host (jump host) quando
  la connessione ha un gateway di tipo SSH_BASTION. Il server apre prima una
  connessione SSH verso il bastion, poi un tunnel TCP verso l'host target,
  e infine stabilisce la sessione SSH finale attraverso il tunnel.
  Se nessun gateway e' assegnato, connessione diretta (comportamento attuale).

  DETTAGLI TECNICI:
  Backend:
    - ssh.service.ts: nuova funzione createSshConnectionViaBastion() che:
      1. Connette al bastion host con credenziali del Gateway (decifrate dalla vault)
      2. Apre un canale TCP forwarded (client.forwardOut()) verso target host:port
      3. Crea un secondo SSH2 Client sullo stream forwarded verso il target
      4. Restituisce la sessione finale (stessa interfaccia SshSession)
    - ssh.handler.ts: controllare se la connessione ha un gateway SSH_BASTION;
      se si', usare createSshConnectionViaBastion() al posto di createSshConnection()
    - SFTP: funziona automaticamente (stesso client SSH2 sottostante)
    - Gestione cleanup: chiudere entrambi i client SSH (bastion + target)
      quando la sessione termina
    - Fallback: nessun gateway = connessione diretta (createSshConnection)
  File coinvolti:
    MODIFICARE: server/src/services/ssh.service.ts
    MODIFICARE: server/src/socket/ssh.handler.ts
    MODIFICARE: server/src/services/connection.service.ts
