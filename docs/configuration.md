## 🎯 Configuration Strategy

Arsenale uses a two-tier configuration approach:

1. **Environment variables** -- Primary source, defined in `.env` at the monorepo root
2. **UI Settings panel** -- Secondary, for values not set via env vars

**When an env var is set:** its value is used as-is, and the corresponding UI field becomes read-only.
**When an env var is unset:** the UI setting is editable, and its value is persisted to the database.

The `.env` file is auto-generated by `make dev` via Ansible templates. The server loads it from `../../.env` relative to `server/src/config.ts`. Never create a separate `server/.env`.
