[x] FOLD-001 — Gestione cartelle per organizzare le connessioni
------------------------------------------------------------------------------
  Priorita: ALTA
  Dipendenze: Nessuna
  Stato backend: COMPLETO (API CRUD gia' implementata)
  Stato frontend: COMPLETO (API client, FolderDialog, ConnectionTree, connectionsStore)
  COMPLETATO: Frontend CRUD completo (API, Dialog, Tree, Store)

  DESCRIZIONE:
  Permettere agli utenti di creare, rinominare ed eliminare cartelle
  direttamente dalla sidebar per organizzare le connessioni in modo
  gerarchico. Il backend supporta gia' nesting illimitato e sortOrder.

  DETTAGLI TECNICI:
  Backend (gia' esistente):
    - GET    /api/folders       → Lista cartelle (ordinate per sortOrder, nome)
    - POST   /api/folders       → Crea cartella (name, parentId?)
    - PUT    /api/folders/:id   → Aggiorna (name?, parentId?)
    - DELETE /api/folders/:id   → Elimina (orfanizza figli e connessioni)
    Modello Prisma: Folder { id, name, parentId, userId, sortOrder }

  Frontend DA IMPLEMENTARE:
    1. Creare client/src/api/folders.api.ts
       - createFolder(name, parentId?), updateFolder(id, data),
         deleteFolder(id), listFolders()
    2. Creare client/src/components/Dialogs/FolderDialog.tsx
       - Campo nome (obbligatorio)
       - Select cartella padre (opzionale, lista cartelle esistenti)
       - Modalita' crea/modifica
    3. Modificare client/src/components/Sidebar/ConnectionTree.tsx
       - Aggiungere bottone "+" accanto a "My Connections" per nuova cartella
       - Context menu su cartelle: Rinomina, Elimina, Nuova sottocartella
    4. Aggiornare client/src/store/connectionsStore.ts
       - Aggiungere azioni: createFolder, updateFolder, deleteFolder
       - Refresh automatico della lista dopo ogni operazione

  FILE COINVOLTI:
    CREARE:  client/src/api/folders.api.ts
    CREARE:  client/src/components/Dialogs/FolderDialog.tsx
