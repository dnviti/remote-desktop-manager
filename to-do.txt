[ ] SMS-034 — Autenticazione MFA via SMS
------------------------------------------------------------------------------
  Priorita: MEDIA
  Dipendenze: 2FA-012 (completato)

  DESCRIZIONE:
  Abilitare l'invio di codici OTP via SMS come metodo di autenticazione
  a due fattori, in alternativa o in aggiunta al TOTP gia' implementato.
  L'utente puo' scegliere SMS, TOTP o entrambi.

  DETTAGLI TECNICI:
  Backend:
    - Integrare un provider SMS (Twilio, AWS SNS, Vonage)
    - Nuove env vars: SMS_PROVIDER (twilio|sns|vonage), credenziali provider
    - Endpoint per: registrazione numero di telefono, invio OTP,
      verifica OTP, rimozione numero
    - Rate limiting per prevenire abusi (costo per SMS)
    - Modello Prisma: aggiungere phoneNumber e smsVerified a User
  Frontend:
    - UI per aggiungere/verificare numero di telefono nelle impostazioni account
    - Step SMS nel flusso di login MFA (scelta tra TOTP e SMS se entrambi attivi)

[ ] OIDC-035 — Supporto provider OAuth2/OIDC generico (Authentik, Keycloak, etc.)
------------------------------------------------------------------------------
  Priorita: MEDIA
  Dipendenze: OAUTH-029 (completato)

  DESCRIZIONE:
  Consentire agli admin di configurare uno o piu' provider OAuth2/OIDC generici
  (es. Authentik, Keycloak, Authelia, Zitadel, qualsiasi IdP OIDC-compliant)
  come metodo di autenticazione. Il sistema usa OIDC Discovery
  (.well-known/openid-configuration) per risolvere automaticamente gli endpoint.

  DETTAGLI TECNICI:
  Backend:
    - Aggiungere strategia passport-openidconnect (o openid-client)
    - Nuove env vars: OIDC_PROVIDER_NAME (nome visualizzato), OIDC_ISSUER_URL,
      OIDC_CLIENT_ID, OIDC_CLIENT_SECRET, OIDC_SCOPES (opzionale, default
      "openid profile email")
    - Auto-discovery endpoint authorize/token/userinfo da OIDC_ISSUER_URL
    - Estendere enum Prisma AuthProvider con valore OIDC (richiede migrazione)
    - Estendere OAuthProfile, VALID_PROVIDERS e SCOPE_MAP nel controller
    - Riutilizzare makeVerifyCallback(), findOrCreateOAuthUser() e OAuth
      service layer esistenti (gia' provider-agnostic)
  Frontend:
    - OAuthButtons.tsx: renderizzare bottone OIDC dinamicamente (nome/icona
      dal endpoint /oauth/providers)
    - LinkedAccountsSection.tsx: supportare visualizzazione provider OIDC
    - UI admin opzionale per configurare nome provider, issuer URL e
      credenziali client (env vars sufficienti per singolo provider)
